5.C, 1.3.1-programmatic-label

Test Name Test ID Test Condition
1.3.1-programmatic-label 5.C The combination of the accessible name, accessible description, and other programmatic associations (e.g., table column and/or row associations) describes each input field and includes all relevant instructions and cues (textual and graphical).

Evaluate Results:

If ANY of the following are TRUE, the content PASSES:

  1. The ANDI Output includes all relevant instructions and cues for the form element, including when fields are required, OR
  2. Descriptive labels and cues are provided by other programmatic associations (e.g., table column and/or row associations), OR
  3. A combination of ANDI Output AND other programmatic association includes all relevant instructions and cues.

Tips:

  • You may need to use other ANDI modules to test for elements such as form fields within a table (table module) or headings (content structure module which we will cover later).
  • If you find a form element and the focusable elements module does not pick up ALL relevant instructions and cues, try using another ANDI module.
  • Test any changes to form elements that occur automatically or as a result of interaction with the page.
  • The content (or option elements) of a drop-down list (or select element) is not part of the form field description, cues, or instructions needed to pass this requirement.
  • Items may have passed or failed 5.A and/or 5.B BUT must still be tested here. (You could have some seemingly contradictory results with these forms tests since each test ID is testing for different disabilities impacted).
  • Pay attention to which ID is testing for what and simply follow the test instructions.
