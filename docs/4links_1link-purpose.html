6.A, 2.4.4-link-purpose

Test Name Test ID Test Condition
2.4.4-link-purpose 6.A The purpose of each link or button can be determined from any combination of the link/button text, accessible name, accessible description, and/or programmatically determined link/button context.

Evaluate Results:

If the following is TRUE, the content PASSES:

  1. The combination of the programmatically determined link/button context and the ANDI Output provide adequate description of the link/button’s purpose.

Interactive Samples

  • Links and Buttons examples

Tips:

  • Considering the surrounding context of the link is different than the previous version of Trusted Tester. It is no longer necessary to evaluate the text of the link or button alone and/or for the link/button text to be entirely unique, as long as the surrounding context provides sufficient detail.
  • It may be necessary to use other ANDI modules and features to successfully complete this test (e.g., tables, structure > lists, structure > headings). For example, headings may also be used to distinguish buttons/links with the same name. Familiarize yourself with the other Test Conditions.
  • When links and buttons change (or their "states" change), use the "Refresh ANDI" button to evaluate the new information for the link/button.
